** 問題
あなたはビットゲームというゲームを考えました。

このゲームは N 人のプレイヤーで行われます。
ゲームの開始時に、各プレイヤーには 1 枚手札が配られ、それとは別に M 枚の山札が用意されます。

手札のカードには、0 または 1 からなる長さ 3 の数字列 p が書かれています。
山札のカードには、0 または 1 からなる長さ 3 の数字列 q および命令 R が書かれています。R は a, b, c のいずれかです。

ゲームは山札のカードが上から順にめくられることで進行します。 山札の各カードには数字列 q および命令 R が書かれており、カードがめくられるたびに全てのプレイヤーは次の操作を行います。

手札の数字列 p とめくられたカードに書かれている数字列 q の同じ桁どうしの数字に対して、命令 R によって新たな数字が生成されます。
こうしてできる長さ 3 の数字列 p' をそのプレイヤーの新たな手札とします。
ここで、命令 R の規則は次のとおりです。

・R = a のとき、1 つでも 1 があれば 1 を生成し、ない場合は 0 を生成する
・R = b のとき、1 つでも 0 があれば 0 を生成し、ない場合は 1 を生成する
・R = c のとき、異なる数字の場合は 1 を生成し、同じ数字の場合は 0 を生成する

全ての山札をめくり終えたとき、最終的な手札の整数が 10 進数表記で一番大きい人が勝利となります。
例えば、プレイヤー 1, 2 の最終的な手札がそれぞれ 101 と 011 であった場合、10 進数表記で 101 > 11 なのでプレイヤー 1 の勝利となります。

プレイヤーの数、各プレイヤーの最初の手札、山札の情報が与えられるので、ゲームの勝者を表示するプログラムを作成してください。

入力例 1 の場合、各プレイヤーの手札と 1 枚目の山札 (q = (1, 0, 0), R = a) に対して、以下のように新しい整数が生成されます。

** 入力される値
入力は以下のフォーマットで与えられます。

N M
p_{1,1} p_{1,2} p_{1,3}
p_{2,1} p_{2,2} p_{2,3}
...
p_{N,1} p_{N,2} p_{N,3}
R_1 q_{1,1} q_{1,2} q_{1,3}
R_2 q_{2,1} q_{2,2} q_{2,3}
...
R_M q_{M,1} q_{M,2} q_{M,3}
・1 行目にプレイヤー数を表す整数 N と、山札の枚数を表す整数 M がこの順で半角スペース区切りで与えられます。
・続く N 行のうちの i 行目 (1 ≦ i ≦ N) には、プレイヤー i の最初の手札である長さ 3 の数字列が与えられます。p_{i, j} (1 ≦ j ≦ 3) は 0 または 1 からなり、プレイヤー i の手札の左から j 番目の数字を表します。
・続く M 行のうちの i 行目 (1 ≦ i ≦ N) の 1 列目には、そのカードに書かれている命令 R_i を表す英小文字 "a", "b", "c" のいずれかが与えられます。j + 1 列目 (1 ≦ j ≦ 3) には、命令が適用される長さ 3 の数字列 q_i の左から j 番目の数字が 0 または 1 で与えられます。
・入力は合計で N + M + 1 行となり、入力値最終行の末尾に改行が 1 つ入ります。

** 期待する出力
最終的な手札の数が 1 番大きかったプレイヤーのID番号を以下の形式で出力してください。
ID_1
ID_2
...
ID_i
...
期待する出力は 1 行以上になります。勝利したプレイヤーのID番号 (1 ≦ i ≦ N) を出力してください。
勝利者が複数いる場合は番号が小さい順で出力してください。末尾に改行を入れ、余計な文字、空行を含んではいけません。

** 入力例1
3 3
1 0 1
1 1 1
0 0 1
a 1 0 0
b 1 1 0
c 0 0 1

** 出力例1
2

** 入力例2
3 2
0 1 0
1 0 1
1 0 0
b 0 0 0
b 0 0 0

** 出力例2
1
2
3
